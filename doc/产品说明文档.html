1. 文档概述

本文档是 model-vram-calc 大模型显存估算系统的完整产品说明文档，面向最终用户、运维人员和技术评估人员编写。文档涵盖产品使用、部署流程、维护指南、计算原理及技术架构五个方面。

本系统为纯前端静态 Web 应用，无需服务器、数据库或任何后端服务，浏览器直接打开即可运行。

2. 用户使用说明

2.1 安装与启动

  1. 从 GitHub Release 页面下载最新版本的 model-vram-calc.zip
  2. 解压 ZIP 文件到本地任意目录
  3. 双击 index.html 文件，系统将在默认浏览器中打开
系统无需安装，解压即用。支持离线运行，不依赖网络连接。

2.2 选择模型

页面顶部提供 12 种主流大模型预设卡片，点击任意卡片即可自动填充该模型的全部参数：

  • DeepSeek 系列：V4-Pro、V4-Flash、V3.2、V3.1、V3、R1
  • GLM 系列：GLM-5、GLM-5.1
  • Qwen 系列：Qwen3.6 35B-A3B、Qwen3 235B、Qwen3.5 235B
  • Minimax：M2.7

选择预设模型后，模型大小、隐藏层维度、层数、Q heads、KV heads 和最大上下文长度将自动填入。

2.3 配置参数

用户可根据需要调整以下参数：

参数说明可选项
模型大小 (GB · FP16)FP16 精度下的模型权重大小任意正整数
量化方式权重量化精度FP16 / BF16 / FP8 / INT8 / FP4 / INT4
Hidden Dim隐藏层维度（embed 维度）任意正整数
LayersTransformer 层数任意正整数
Q Heads注意力头数任意正整数
KV HeadsGQA KV 头数任意正整数
上下文长度输入 + 输出 token 总数16K / 32K / 64K / 128K / 200K / 320K / 1M
KV Cache 格式KV Cache 精度FP16 / FP8 / INT4-FP4

2.4 GPU 配置

提供 7 种 GPU 预设芯片，点击即可自动设置单卡显存：

  • NVIDIA 数据中心：H200 141G、H20 141G、B200 192G
  • NVIDIA 消费级：RTX 5090、RTX 4090
  • 阿里 PPU：A710E、真武 810E

除预设外，用户可手动调整 GPU 数量、系统预留显存和推理开销比例。

2.5 查看结果

点击「重新计算」按钮后，结果区域将展示 6 项核心指标：

  • 模型权重显存 — 量化后权重占用的显存（GB）
  • KV Cache / Session — 单个并发 session 的 KV Cache 显存
  • 单 Session 总显存 — 权重 + KV Cache + 推理开销
  • 最大并发 Session — 集群支持的最大并发数
  • 集群总显存 — GPU 数量 × 单卡显存
  • 最少 GPU 数量 — 加载权重所需的最少 GPU 卡数

底部「计算明细」区域提供完整的公式推导过程，右侧附带微信赞赏码（点击可放大查看）。

2.6 常见问题

Q：页面打不开怎么办？
A：请确保使用现代浏览器（Chrome 90+、Edge 90+、Safari 14+、Firefox 88+），检查是否已解压 ZIP 文件后再打开。

Q：计算结果准确吗？
A：本工具基于业界通用的显存估算公式，计算结果可作为部署前的参考预估值，实际显存占用可能因推理框架实现细节略有差异。

Q：如何获取最新模型参数？
A：模型预设参数来源于官方技术报告，如需最新数据，请参考模型官方文档。

3. 系统部署说明

3.1 获取软件

通过以下两种方式获取软件：

  • Release ZIP — 访问 GitHub Releases 页面，下载 model-vram-calc.zip（仅包含运行必须的文件）
  • 源码克隆 — git clone https://github.com/soongzx/model-vram-calc.git

3.2 本地部署

  1. 将 model-vram-calc.zip 解压到目标目录
  2. 双击 index.html 即可使用

目录结构如下：

model-vram-calc/
├── index.html          # 主程序入口
├── vram_calc.html      # 计算公式说明页
└── donate/
    ├── about.html      # 关于页面
    └── 503160023536.jpg  # 微信赞赏码

3.3 服务器部署

如需部署为 Web 服务供团队访问，可使用任意静态文件服务器：

# Python
python3 -m http.server 8000

# Node.js (npx)
npx serve . -p 8000

# Nginx
server {
    listen 80;
    root /path/to/model-vram-calc;
    index index.html;
}

3.4 CI/CD 自动发布

项目内置 GitHub Actions 工作流（.github/workflows/release.yml），每次推送到 main 分支时自动：

  1. 从 index.html 提取版本号
  2. 打包运行必须的文件为 ZIP
  3. 创建 GitHub Release 并上传 ZIP 包

5. 计算方式说明

5.1 模型权重显存

权重显存(GB) = FP16 模型大小(GB) × 量化压缩比
量化方式bytes/参数压缩比
FP16 / BF1621
FP8 / INT810.5
FP4 / INT40.50.25

5.2 KV Cache 显存

KV Cache(GB) = 2 × numLayers × (hiddenDim × kvHeads/attnHeads) × contextLen × bytesPerElem / 1e9
其中 kvHeads / attnHeads 为 GQA KV 头比例。非 GQA 模型（MHA）中 kvHeads = attnHeads，比例为 1。
格式bytes/elem说明
FP162传统默认，无损
FP81vLLM/SGLang 主流
INT4/FP40.54-bit 量化，相对 FP8 再减半

5.3 推理开销

开销(GB) = 权重显存(GB) × 开销比例
场景建议比例
高度优化（vLLM PagedAttention）8%
vLLM 默认12%
SGLang15%
保守估计20%

5.4 集群与并发计算

集群总显存(GB) = 单卡显存 × GPU 数量
可用显存(GB)   = 集群总显存 - 系统预留
最少 GPU 数    = ceil((权重显存 + 预留) / 单卡显存)
最大并发       = floor((可用显存 - 权重显存) / (KV Cache + 开销))
权重只加载一份，所有 session 共享。并发容量取决于剩余显存能容纳多少份独立的 KV Cache + 推理开销。

6. 系统技术白皮书

6.1 系统架构

model-vram-calc 采用极简的单页应用（SPA）架构，所有代码集中在一个 HTML 文件中，无任何外部依赖。

┌─────────────────────────────────────────┐
│           index.html (主程序)            │
│  ┌─────────┬─────────┬──────────────┐  │
│  │ HTML    │ CSS     │ JavaScript   │  │
│  │ 结构层  │ 样式层  │ 逻辑层       │  │
│  └─────────┴─────────┴──────────────┘  │
└─────────────────────────────────────────┘
┌──────────────────┐    ┌───────────────┐
│ vram_calc.html   │    │ donate/       │
│ 计算公式说明     │    │ about.html    │
│                  │    │ 503160....jpg │
└──────────────────┘    └───────────────┘

6.2 设计决策

  • 单文件架构 — 避免多文件依赖，降低分发复杂度，确保离线可用
  • 无构建步骤 — 无需 Webpack/Vite 等工具，编辑后即可生效
  • 原生 JavaScript — 不引入 React/Vue 等框架，减少体积和加载时间
  • CSS 变量与响应式 — 通过 4 个断点覆盖桌面、平板、手机全终端
  • 暗色主题 — 面向开发者用户，采用玻璃拟态（glassmorphism）风格

6.3 性能与安全

性能：页面体积约 25KB（gzip 后更小），首屏加载时间 < 100ms。计算逻辑为纯 CPU 运算，响应时间 < 10ms。无任何网络请求，不存在首屏阻塞。

安全：纯客户端应用，不收集、不传输任何用户数据。无 Cookie、无追踪代码、无第三方脚本。所有计算在本地浏览器中完成。

6.4 未来规划
